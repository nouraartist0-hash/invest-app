# واجهة التطبيق — النسخة الأولى (Frontend فقط)

## شنو موجود بهذا الملف
- 19 شاشة HTML/CSS مستقلة (بدون build tools)، بتصميم موبايل موحّد.
- افتح `index.html` بالمتصفح للتصفح، أو افتح أي شاشة من مجلد `screens/` مباشرة.
- كل الشاشات تتنقل بينها فعلياً (روابط شغالة) بس بدون تخزين بيانات حقيقي — كل قيمة بالشاشات (الأرصدة، الأسماء، النسب) نموذج توضيحي (mock).
- نظام التصميم (الألوان/الخطوط/المكونات) موجود بـ `assets/style.css` تقدر تعدل عليه بمكان وحد وينعكس على كل الشاشات.

## شنو مو موجود (ومهم تعرفه)
- **ما فيه ربط بقاعدة بيانات حقيقية.** ما أقدر من هذا الطرف أوصل تلقائياً بمشروع Firebase/Supabase الحقيقي مالتك وأربط تسجيل الدخول، الإيداع، السحب، إلخ — هذا يحتاج مفاتيح مشروعك الفعلية (API keys) وإعداد قواعد الأمان (security rules)، وهذا نسويه سوا بجلسة منفصلة.
- **ما فيه معالجة دفع حقيقية (USDT/TRC20).** ربط محفظة حقيقية لاستقبال/إرسال عملات رقمية يحتاج تكامل مع مزود دفع أو عقدة بلوكتشين، وهذا قرار تقني وقانوني منفصل لازم تاخذه بعناية (رسوم، أمان، توافق قانوني بالعراق).
- **ما فيه لوحة تحكم إدمن كاملة.** أضفتها لاحقاً كخطوة منفصلة إذا تريدها.

## الخطوة الجاية المقترحة
1. تراجع الشاشات وتعطيني ملاحظات (تعديل نصوص، ألوان، ترتيب).
2. نجهز مشروع Firebase/Supabase حقيقي (Auth + Database) ونربطه بشاشة تسجيل الدخول والحساب كبداية.
3. نبني شاشة السحب/الإيداع بمنطق حقيقي (مراجعة يدوية أو مزود دفع) بعد ما يكون عندك قرار واضح بآلية استلام وصرف الأموال.
4. لوحة تحكم الإدمن (لإدارة المستخدمين والعروض والمعاملات).

**ملاحظة:** العائد بكل الشاشات مصمم كنطاق تقديري مبني على الأداء الفعلي (مو نسبة ثابتة مضمونة)، متل ما اتفقنا — خلّي هذا المبدأ ثابت بأي تعديل جاي حتى لو غيرت الأرقام أو التصميم.
